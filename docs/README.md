# 文献检索与芝加哥引用助手

## 项目简介

本项目提供一个可扩展的文献搜索与过滤平台，支持从多个数据源（如 Crossref 与 arXiv）检索出版物，并根据 DOI →（标题+年份）→ URL 哈希 的优先级策略进行去重。系统会把去重结果记录到 `data/seen.sqlite` 中，以避免重复成果的再次呈现。

借助可视化的 Web UI，用户可以：

- 按关键词、作者、时间范围、语言、格式类型、范围等维度组合检索（支持多选/全选）。
- 自由选择抓取范围：可以勾选指定数据源或启用“使用所有数据源”。
- 管理数据源：可添加新的 Crossref / arXiv 配置并永久保存。
- 查看包含完整文献信息与访问链接的检索结果。
- 在结果右侧多选文献，一键复制芝加哥格式引用到剪贴板或导出为 `.txt` 文件。

## 安装与启动

1. **安装依赖**

   ```bash
   python -m venv .venv
   source .venv/bin/activate  # Windows 使用 .venv\\Scripts\\activate
   pip install -r requirements.txt
   ```

2. **启动服务**

   ```bash
   flask --app app run
   ```

   或使用 Python 直接运行：

   ```bash
   python app.py
   ```

   默认会在 `http://127.0.0.1:5000/` 提供 Web UI。

   > 💡 `python app.py` 默认关闭自动重载，以避免在 Windows + VS Code Debugger 环境下出现 `socket.fromfd` 错误。
   > 如需带调试重载的体验，请使用 `flask --app app run --debug`。

## 使用指南

### 1. 配置检索条件

- **关键词**：支持多个关键词，空格分隔，并可通过“关键词搜索模式”选择匹配正文、关键词/摘要字段或仅标题。
- **作者**：多个作者以逗号分隔。
- **时间范围**：选择开始与结束日期。
- **语言 / 格式 / 范围**：语言与格式采用“胶囊”式多选组件，支持全选/清空，同时会在标签下方实时展示已选数量。范围可输入出版类型、出版方或来源名称，每行一个。
- **抓取范围**：默认勾选“使用所有数据源”。取消勾选后，可在数据源列表中多选。

### 2. 执行检索

点击“执行检索”，系统会向所选数据源发起请求，并在去重后展示结果。每条记录包含：

- 标题
- 作者
- 年份
- 出版方/来源
- 类型
- DOI（若存在）
- 摘要摘要（前 220 个字符）
- 访问链接

### 3. 生成芝加哥引用

- 在结果列表中勾选任意条目或点击“全选结果”。
- 右侧实时展示所选条目的芝加哥引用文本。
- 点击“复制到剪贴板”即可复制。
- 点击“导出为TXT”将引用保存到 `data/export.txt` 并在界面显示导出路径。

### 4. 数据源管理

在“数据源管理”区域可以新增数据源：

1. 输入名称（如“Crossref 扩展”）。
2. 选择类型（目前支持 `crossref` 与 `arxiv`）。
3. 如需自定义参数（例如 Crossref 的 `rows`），可在 JSON 配置框中填入。
4. 点击“新增数据源”后，配置会被写入 `data/sources.json` 并立即可用。

### 5. 去重策略

系统按以下优先级去重，遇到重复即忽略：

1. **DOI** 完全相同。
2. **标题 + 年份** 相同。
3. **URL 哈希** 相同。

所有已出现的记录均会写入 `data/seen.sqlite`，确保后续检索不会重复呈现。

## 进阶用法

- 若需要扩展新的数据源，可在 `literature_harvester/sources/` 下新增模块，并在 `registry.register()` 中注册。
- 后端导出的芝加哥格式生成逻辑位于 `literature_harvester/formatting.py`，可按需调整格式化规则。

## 常见问题

1. **检索不到结果？**
   - 检查关键词是否过于严格，或尝试放宽时间/语言/格式限制。
   - 确保数据源配置正确且目标 API 可访问。
   - 若启用了语言过滤，请确认所选语言与数据源支持的语言一致。内置选项涵盖 `English / 英语`、`中文`、`Español / 西班牙语`、`Deutsch / 德语`、`Français / 法语`、`Русский / 俄语`、`日本語 / 日语`、`العربية / 阿拉伯语`、`فارسی / 波斯语` 与 `Türkçe / 土耳其语`，系统会自动匹配 Crossref 的 ISO 代码。若需检索所有语言请取消选择。

2. **无法访问外部 API？**
   - 确认本机网络可访问 `api.crossref.org` 与 `export.arxiv.org`。
   - 若需要代理，可在系统级设置或自定义源配置中补充参数。

3. **关键词搜索为何会出现作者匹配？**
   - 默认的“正文内容匹配”会同时搜索标题、摘要与主题词，可能包含作者信息。
   - 如需严格限制，请在“关键词搜索模式”下选择“关键词/摘要字段”或“仅标题”以过滤掉作者命中的结果。

4. **如何清空去重记录？**
   - 删除 `data/seen.sqlite` 即可。重启服务后会自动重建空表。
